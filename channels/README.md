# 渠道接入中心

如果你已经把 OpenClaw 部署好了，下一步就从这里开始，把它接到真实使用的聊天和协同入口。

本目录主要解决两件事：

- 把 OpenClaw 接到国内常用渠道
- 明确每条渠道的能力边界、接入方式和验证步骤

## 当前可用

- 飞书：看 [飞书接入对话（基础篇）](./02-feishu-basic.md)
- 企业微信：看 [企业微信智能机器人接入](./01-wecom-intelligent-bot.md)

## 目录约定

后续这里会逐步拆成两类：

- 原生渠道：OpenClaw 官方已经支持的渠道
- 桥接渠道：通过平台机器人、Webhook 或自建适配层接入的渠道

## 建议阅读顺序

1. 先完成 [部署与安装中心](../deploy/README.md)
2. 再选择 1 个渠道接入
3. 跑通第一个真实对话或推送
4. 回到 [5分钟快速上手](../quickstart/00-5min-quickstart.md)

## 接下来会补什么

- 飞书高级篇：多机器人、多账号、多 Agent 绑定
- 飞书高级篇：会话隔离、`dmScope`、routing / binding
- 钉钉接入
- QQ 接入

## 说明

- 飞书基础篇优先采用飞书官方的 OpenClaw 专页作为入口。
- 企业微信这篇文档基于企业微信官方帮助中心整理，并将截图落到了仓库本地。
