# 想法 → 实验 → 决策（ADR）夜间闭环

> 白天记录想法，夜间自动实验，次日用 ADR 结构沉淀决策。

## 这个案例能帮你做什么

- 把“想到就忘”变成有节奏的决策生产线。
- 通过子代理自动跑探索、调研、编码实验。
- 用 ADR 风格记录决策背景、备选方案和结论。

## 你需要的 Skills（按类型）

| 类型 | Skill / 工具 | 用途 | 来源 |
|---|---|---|---|
| 外部 | `log skill` | 记录想法与任务状态 | log skill |
| 内置 | `cron job` | 夜间挑选任务并触发实验 | OpenClaw Built-in |
| 内置 | `scientist subagents` | 调研/实验并行执行 | OpenClaw Built-in |
| 外部 | `ADR 记录结构` | 沉淀决策过程 | ADR |

## 快速体验版（先跑一轮）

1. 白天先记录 3 条想法任务。
2. 夜间只跑 1 条任务实验。
3. 次日输出 1 条 ADR 决策记录。

```text
请把今天记录的任务中选一条执行夜间实验：
- 输出实验过程与结果
- 给出可选方案对比
- 生成一份 ADR 决策记录草稿
```

## 稳定自动版（可长期运行）

### 推荐流程

1. 白天采集想法并结构化为 TASK。
2. 夜间自动挑选任务并并行实验。
3. 早晨评审结果，决定继续跟进或定稿。

## 成功标准

- [ ] 每天都有可审阅的实验结果。
- [ ] 关键决策都有 ADR 记录。
- [ ] 决策链路从想法到结论可追溯。
