# Telegram 远程站点迁移（Notion → Astro + DNS）

> 不开电脑，仅通过 Telegram 驱动站点迁移和域名切换。

## 这个案例能帮你做什么

- 把内容迁移、静态站重建、DNS 切换串成一个远程流程。
- 在碎片时间完成完整迁移任务。
- 提高“移动端发起→自动执行→结果回传”的可行性。

## 你需要的 Skills（按类型）

| 类型 | Skill / 工具 | 用途 | 来源 |
|---|---|---|---|
| 内置 | `Notion` | 旧内容来源 | Notion |
| 外部 | `Astro` | 新站结构与页面承载 | Astro |
| 外部 | `Cloudflare` | 域名切换与 DNS 管理 | Cloudflare |
| 内置 | `Telegram` | 远程指令与状态回传 | OpenClaw Built-in |

## 快速体验版（先跑一轮）

1. 先迁移 1-2 篇内容验证流程。
2. 输出迁移差异（链接、图片、排版）。
3. 确认后再批量迁移。

```text
请先做最小迁移：
- 从 Notion 导入 2 篇文章到 Astro
- 检查链接与媒体资源是否完整
- 生成迁移报告并给出批量迁移计划
```

## 稳定自动版（可长期运行）

### 推荐流程

1. 先内容迁移，再 DNS 切换。
2. DNS 切换前做可访问性检查。
3. 切换后做回归检查（404、资源加载、SEO 基础项）。

## 成功标准

- [ ] 迁移前后内容完整度可核对。
- [ ] DNS 切换后访问稳定。
- [ ] 整个迁移过程有可复用步骤。
