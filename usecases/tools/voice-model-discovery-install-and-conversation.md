# 语音模型发现、安装并接入对话

> 发现一个新的 voice model 后，直接让 OpenClaw 负责检查、安装并开始和你对话。

## 这个案例能帮你做什么

- 把“看到一个新语音模型”变成“立即验证能不能装、装完能不能说”。
- 省掉手动查资料、切环境、自己做第一轮联调的过程。
- 让 OpenClaw 从模型检查一路走到可对话验证，而不是只停在安装层面。

## 你需要的 Skills（按类型）

| 类型 | Skill / 服务 | 用途 | 来源 |
|---|---|---|---|
| 外部（按需） | `voice model` | 作为需要检查、安装并接入的目标模型 | 来源原帖 |

## 快速体验版（先跑一轮）

1. 先提供 voice model 的名称、链接或说明。
2. 本轮目标只做到“能否安装、能否接入、能否说一句话”。
3. 失败也没关系，关键是把卡点说清楚。

```text
你是我的 OpenClaw 助手。
我刚发现一个新的 voice model，请你：
1. 先检查它是否适合当前环境。
2. 如果可用，完成安装并接入对话。
3. 安装后立刻用它跟我说一句话，验证已经能正常工作。
4. 全程把每一步结果回报给我；如果失败，明确卡在哪一步。
```

## 稳定自动版（可长期运行）

### 推荐流程

1. 先让 OpenClaw 检查这个 voice model 是什么、是否适合当前环境。
2. 确认可行后，再进入安装步骤，而不是一上来就盲装。
3. 安装完成后，立刻把它接入对话链路，验证“不只是装好了，而是真的能说话”。
4. 如果验证失败，要求 OpenClaw 明确告诉你失败发生在检查、安装还是接入阶段。

## 成功标准

- [ ] 新发现的 voice model 能快速完成可行性判断。
- [ ] 安装完成后，能立刻进入一次实际对话验证。
- [ ] 失败时能明确知道具体卡在哪一步。
