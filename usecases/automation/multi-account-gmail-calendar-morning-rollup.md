# 多账号 Gmail + Calendar 晨间汇总

> 用 `gog` 把多个 Gmail 和 Calendar 账号合成一份晨间总览，起床后先看重点而不是逐个点开。

## 这个案例能帮你做什么

- 把多个账号昨夜收到的邮件和今天的日程汇总成一份晨报。
- 控制每次展示多少封邮件、多少个事件，避免信息量失控。
- 按你的习惯设置运行频率，不必手动轮流检查多个账号。

## 你需要的 Skills（按类型）

| 类型 | Skill / 服务 | 用途 | 来源 |
|---|---|---|---|
| 外部 | [`gog`](https://clawhub.ai/steipete/gog) | 读取 Gmail 和 Calendar 中的邮件与日程 | ClawHub |
| 外部（按需） | `morning rollup skill` | 把多个账号的内容整理成晨间汇总 | 来源原帖 |

## 快速体验版（先跑一轮）

1. 先接入你最常用的 2 个账号，避免第一次信息量太大。
2. 本轮先只做“昨夜邮件 + 今天日历”的合并查看。
3. 先输出晨报，不做回复、归档或写回操作。

```text
你是我的 OpenClaw 助手。
请用 `gog` 帮我做一版晨间汇总：
1. 读取我所有已接入的 Gmail 和 Calendar 账号。
2. 总结昨夜新邮件和今天日程。
3. 每个账号分别列出，并限制展示数量。
4. 本轮先只输出晨报，不做回复、归档或写入。
```

## 稳定自动版（可长期运行）

### 推荐流程

1. 用 `gog` 接入多个 Gmail 和 Calendar 账号。
2. 设置固定的晨间运行时间，让 OpenClaw 在你开始一天之前把信息收拢好。
3. 根据你的偏好配置每次展示多少封邮件、多少个事件。
4. 如果账号较多，优先保证“重点账号先看、次要账号少看”，避免晨报过长。

## 成功标准

- [ ] 多个 Gmail 和 Calendar 账号能被稳定汇总到同一份晨报里。
- [ ] 邮件和日程的展示数量可以按需控制。
- [ ] 晨报频率和内容密度能够长期保持可读。
