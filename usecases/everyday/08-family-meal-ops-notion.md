# 家庭餐食运营系统（Notion）

> 把“做饭安排、采购、天气判断”合并到一个家庭协作系统。

## 这个案例能帮你做什么

- 提前规划全年/每周餐食，减少临时决策成本。
- 自动生成按商店和货架分组的采购清单。
- 将天气信息与餐食安排联动（如烧烤/汤类时机）。

## 你需要的 Skills（按类型）

| 类型 | Skill / 工具 | 用途 | 来源 |
|---|---|---|---|
| 内置 | `Notion` | 承载餐食模板与清单 | Notion |
| 外部 | `天气信息` | 调整当日餐食策略 | Weather API |
| 内置 | `提醒与调度` | 晨间/晚间提醒计划晚餐与采购 | OpenClaw Built-in |

## 快速体验版（先跑一轮）

1. 先做 1 周版本（不是全年）。
2. 建立 3 个基础库：菜谱、采购、排班。
3. 让 OpenClaw 每天提醒一次。

```text
请为我们做一个 7 天家庭餐食计划：
- 每天 1 个晚餐方案
- 自动生成采购清单（按商店分类）
- 标注适合天气条件
- 每天早晚提醒一次
```

## 稳定自动版（可长期运行）

### 推荐流程

1. 扩展到月度/年度模板。
2. 固化常用菜谱与主厨分工。
3. 每周复盘“采购命中率”和“临时改菜原因”。

## 成功标准

- [ ] 晚餐计划与采购清单同步更新。
- [ ] 天气变化会触发合理调整。
- [ ] 家庭成员分工清晰、执行稳定。
