# 伴侣双 Bot 共享记忆层（shared-memory）

> 两个 OpenClaw 通过共享记忆层同步关键上下文，实现跨人协作。

## 这个案例能帮你做什么

- 把“我知道但对方 Bot 不知道”的信息差降到最低。
- 在伴侣/合伙协作场景中共享必要状态。
- 支持“选择性共享”，避免全部记忆无差别暴露。

## 你需要的 Skills（按类型）

| 类型 | Skill / 工具 | 用途 | 来源 |
|---|---|---|---|
| 外部 | `shared-memory skill` | 跨 Bot 同步上下文和状态 | shared-memory |
| 内置 | `OpenClaw 双实例` | 各自执行并共享必要信息 | OpenClaw Built-in |

## 快速体验版（先跑一轮）

1. 先定义“共享字段”（如项目状态、待办、日程）。
2. 让两边 Bot 只同步这部分字段。
3. 验证同步后是否能正确接力任务。

```text
请建立一套共享记忆规则：
- 只共享项目状态、待办和时间节点
- 私人敏感内容不共享
- 每次同步后给出差异摘要
```

## 稳定自动版（可长期运行）

### 推荐流程

1. 共享白名单字段固定化。
2. 变更记录带时间戳与来源 Bot。
3. 定期清理过期共享状态。

## 成功标准

- [ ] 关键协作信息可双向同步。
- [ ] 共享范围可控，不越界。
- [ ] 任务交接时信息完整且一致。
